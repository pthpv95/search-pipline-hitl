# PLAN_FINAL Priorities

## Must-Have

- Stable typed state and graph contracts
- Explicit `dev`, `live`, and `eval` modes
- Stubbed end-to-end graph in Week 1
- Real search integration with `dev`-only mocks
- Controlled search loop with `max_loops`
- Non-interactive end-to-end pipeline
- Saved run artifacts
- Citation integrity checks
- Human review with documented checkpoint model
- Honest eval script on real eval runs
- Basic timing, token, and cost tracking
- README with accurate setup and limitations

## Nice-to-Have

- Fancy terminal formatting in `cli_review.py`
- Multiple search providers in Week 2
- Reachability checks for every saved URL
- Richer report formatting beyond markdown basics
- More than one report template tone
- Extra eval heuristics beyond citation integrity and coverage
- Demo polish for portfolio presentation
- Additional convenience scripts

## Not Worth Optimizing Early

- Exact token totals in sample output
- Exact loop counts for sample topics
- Fixed test count targets
- Perfect prompt wording before interfaces are stable
- Extensive README storytelling before the pipeline works
